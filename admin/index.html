Help

Signing In

There are two ways in — use whichever one you were given.

A
Shared Password
A single password shared by all staff, entered under Password on the sign-in screen. Quick, but not tied to your identity — publishing this way saves drafts locally and needs a manual export/push to go live. Ask whoever manages the site for the current password.
B
Netlify Identity (recommended)
A real per-person account, entered under Netlify Identity Email/Password. Publishing this way goes live immediately — no extra steps. You need to be invited first (see "Setting Up a New Login" below). Forgot your password? Click Forgot password? on the sign-in screen and check your email.

Publishing an Event Article

1
Open the editor
Click Event in the sidebar, then + New Article (or Edit on an existing one).
2
Fill in the basics
At minimum: Publish Date, Category, and Title (Chinese) — the English title is optional but recommended. Leave Summary blank to auto-generate one from the body.
3
Add a featured image
Click Upload next to Featured Image. It appears on the news list and at the top of the article.
4
Build the article body
Use + Text Block for writing (with a bold/italic/heading/list/link toolbar) and + Image Block for photos with their own alignment, size, and caption. Reorder blocks with the ▲/▼ arrows.
5
Preview, then save
Click Preview any time to see how it will actually look. When ready, click Save Article.
6
Publish
Go to Publish in the sidebar and click Publish to Website. If you're signed in with Netlify Identity, this goes live right away. On the shared password, this saves locally — see the Publish panel for how to get it live.

Getting a Personal Access Token

Needed once to connect the Inbox tab (so it can read Contact/Membership form submissions) and to enable email notifications for new submissions.

1
Log in to Netlify
Go to app.netlify.com and sign in with the account that manages this site.
2
Open User settings
Click your avatar in the top-right corner, then User settings.
3
Go to Personal access tokens
In the left sidebar, click Applications, then find the Personal access tokens section and click New access token.
4
Name it and generate
Give it a description like "CCCWA Admin Inbox" and click Generate token.
5
Copy it immediately
Netlify shows the token only once. Copy it right away — if you lose it, you'll need to generate a new one.
6
Paste it into the Inbox tab
Back in this admin panel, go to Inbox, paste the token into Personal Access Token, also fill in the Site ID (Netlify → Site configuration → General → Site details), then click Save & Connect.
Treat this token like a password — anyone with it can access your Netlify account's data. It's stored only in your browser's local storage, never sent anywhere except Netlify's API.

Setting Up a New Login (Netlify Identity)

For whoever manages the Netlify account — do this once per new staff member.

1
Open Netlify's dashboard
Go to app.netlify.com, select this site, then Site configuration → Identity → Users.
2
Invite the person
Click Invite users and enter their email address. Netlify emails them an invite link.
3
They click the link
It opens this admin panel with a Set Your Password screen. They pick a password, and they're logged straight in.
4
Done
From then on, they sign in with their email + that password under Netlify Identity Email/Password on the sign-in screen — no need to be re-invited.
If an invite link ever seems "broken" (no set-password form appears), the token has likely expired or was already used once before — delete the pending user under Identity → Users and send a fresh invite.

What's New: Inbox, Trash & Homepage Editing

A reference for the newer features added to this admin panel — what each one does and how it fits together.

Inbox

The Inbox tab shows Contact/Membership form submissions as a thin, Gmail-style list — sender, subject, and a message preview on one line, with a date/time on the right.

  • Click a row to open the full message. Opening it automatically marks it read.
  • Each row has a trash-can button on the far left (delete without opening) and a checkmark on the right (mark read without opening).
  • Multiple notification emails can be connected at once under the Netlify Forms Connection box — add as many as you want, each shown in its own row with its own Remove button.
  • The "Show raw data" link inside an open message reveals exactly what Netlify received for that submission — useful if a field (like email or subject) doesn't look right, since it shows the raw values Netlify actually stored.

Trash

Deleting a news article, event, or inbox message no longer removes it right away — it moves to the Trash tab first, so mistakes are recoverable.

  • News/Events: a trashed item is dropped from what gets published, but stays recoverable in Trash until you either Restore it or Delete Permanently.
  • Inbox messages: trashing one only hides it locally — Netlify still has it. Restore brings it back to the Inbox; Delete Permanently is the step that actually removes it from Netlify for good (needs your Netlify token connected).
  • Empty Trash permanently deletes everything currently in Trash in one go — use it carefully, since that step can't be undone.

Homepage & Mission Content

The Homepage tab (next to Dashboard) makes the site's core wording editable — no code required. It's saved through the same Save → Publish flow as News/Events, backed by content/homepage.json.

  • Hero Section: the big title and subtitle at the very top of the homepage.
  • About Section: the "Who We Are" tag, title, and both paragraphs on the homepage.
  • Footer Links: the link row at the bottom of every page next to the logo (Facebook, etc.) — add, remove, or reorder any number of links. The old "Original Site" link was removed from here.
  • Programs & Services: the 6 cards on the homepage — add a new one, remove one, reorder them, and edit each card's icon, title, description, and link.
  • Latest News on Homepage: defaults to Automatic (always shows the 3 newest posts). Switch to Manual to hand-pick specific posts and their order instead — useful for pinning something important above newer posts.
  • Mission Page: the hero title/subtitle, "About Us" text, the numbered Objectives list (add/remove/reorder), and the Values text on the separate Mission page.
Remember to click Save Changes at the top or bottom of the Homepage tab first, then Publish from the sidebar to make edits live — saving alone only stores the draft in this browser.
